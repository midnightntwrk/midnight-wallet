### Version 3.0.0

**Version:** 3.0.0 **Date:** 2026-03-20 **Environment:** Mainnet, Preprod, Preview

---

### High-level summary

Major release focused on accurate fee estimation and developer ergonomics. Fee calculation is split into a context-free
`calculateFee` and a wallet-aware `estimateFee` that accounts for the coins available in the wallet and the balancing
transaction cost. The SDK is updated to ledger v8. A new `fetchTermsAndConditions` API lets wallet builders retrieve
network Terms and Conditions before initialization. A promise-based `QueryRunner` simplifies GraphQL queries for
developers not using Effect.

---

### Audience

This release is relevant for developers who:

- Build wallets for the Midnight network
- Integrate Midnight token transfers into their DApps
- Need to manage shielded (private) and unshielded token balances
- Implement atomic swaps between parties

---

### What changed (Summary of updates)

- Fee calculation split into `calculateFee` (transaction-only) and `estimateFee` (wallet-aware, includes balancing cost)
- Updated to ledger v8 across the SDK
- New `WalletFacade.fetchTermsAndConditions()` static method for retrieving network T&C
- New `QueryRunner.runPromise()` utility for executing GraphQL queries as plain Promises
- `SecretKeysResource` in shielded/dust wallet now clears keys from memory after use
- Shielded wallet now clears pending coins on transaction failure
- Improved error messages for invalid transaction submission and balancing failures

### New features

**Terms and Conditions Retrieval**

**Description:** New static method `WalletFacade.fetchTermsAndConditions(configuration)` retrieves the current Terms and
Conditions (URL and SHA-256 hash) from the network indexer. Can be called before or independently of wallet
initialization. Accepts any configuration that includes `indexerClientConnection.indexerHttpUrl`, so the shared wallet
configuration can be passed directly.

---

**Promise-based QueryRunner**

**Description:** New `QueryRunner.runPromise()` utility in `@midnight-ntwrk/wallet-sdk-indexer-client` that executes
GraphQL queries as plain Promises without Effect boilerplate. Takes a query, variables, and server config and returns a
`Promise<R>`.

### Breaking changes or required actions for developers

**Ledger v8**

The SDK is updated to ledger v8 across all packages. Developers must update their ledger dependency to v8.

---

**Dynamic Fee Calculation (`@midnight-ntwrk/wallet-sdk-facade`, `@midnight-ntwrk/wallet-sdk-dust-wallet`)**

The previous `calculateFee` method attempted to estimate the total fee a wallet would pay, but did so inaccurately
because it could not account for the coins available in the wallet or the cost of the balancing transaction. In v3.0.0,
fee computation is split into two distinct methods:

- **`calculateFee`** — computes the fee for a given transaction in isolation. It does not consider the wallet's current
  coin set or the additional cost of a balancing transaction. Use this when you need a quick, context-free fee figure.
- **`estimateFee`** — computes the realistic total fee the _current wallet_ would pay, factoring in which coins are
  available for fee payment and the cost of the balancing transaction itself. Because the fee depends on the coins
  selected, and different coin sets produce different transaction sizes, this method requires a secret key and wallet
  state. Throws `InsufficientFundsError` if the wallet cannot cover the fee.

**WalletFacade migration:**

```ts
// Before (v2.0.0) — single method
const fee = await wallet.calculateTransactionFee(tx);

// After (v3.0.0) — two methods

// Fee for the transaction only (no balancing costs, no wallet context):
const txFee = await wallet.calculateTransactionFee(tx);

// Total fee the wallet would actually pay (includes balancing tx, wallet-aware):
const totalFee = await wallet.estimateTransactionFee(tx, secretKey, {
  ttl: new Date(/* ... */), // optional
  currentTime: new Date(), // optional
});
```

**DustWalletAPI migration:**

```ts
// Before — single calculateFee
const fee = await dustWallet.calculateFee(transactions);

// After
// Transaction fee only (context-free):
const txOnlyFee = await dustWallet.calculateFee(transactions);

// Total fee the wallet would pay (wallet-aware, may throw InsufficientFundsError):
const totalFee = await dustWallet.estimateFee(secretKey, transactions, ttl, currentTime);
```

---

**`CoinSelection` type change (`@midnight-ntwrk/wallet-sdk-dust-wallet`)**

```ts
// Before — returned multiple coins summed to a target
type CoinSelection<T> = (coins: readonly CoinWithValue<T>[], target: bigint) => CoinWithValue<T>[];

// After — returns a single coin (smallest available) or undefined
type CoinSelection<T> = (coins: readonly CoinWithValue<T>[]) => CoinWithValue<T> | undefined;
```

---

**`InsufficientFundsError` (`@midnight-ntwrk/wallet-sdk-dust-wallet`)**

New error type thrown by `estimateFee` when the wallet cannot cover the fee. Includes `message` and `tokenType` fields.

```ts
import { InsufficientFundsError } from '@midnight-ntwrk/wallet-sdk-dust-wallet';
```

### Known issues

**Issue:** Transaction history not implemented for shielded and dust wallets

**Description:** The shielded and dust wallets do not track transaction history. The shielded wallet's transaction
history getter throws "not yet implemented". Only the unshielded wallet maintains transaction records.

### Fixed defect list

- Shielded wallet now correctly clears pending coins when a transaction fails. Previously, coins marked as pending were
  not released on failure, causing incorrectly reduced available balances until wallet restart.
- `SecretKeysResource` in shielded and dust wallets now securely clears cryptographic keys from memory after use,
  instead of only nullifying the reference.
- Node client returns clearer error messages when transaction submission fails due to an invalid transaction.
- Dust wallet reports more descriptive error messages when balancing fails.
- Dev-only dependencies moved out of production dependencies in `@midnight-ntwrk/wallet-sdk-utilities`.
